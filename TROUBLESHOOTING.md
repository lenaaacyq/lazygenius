# 故障复盘：从 401 到写库失败的连锁问题

## 背景
一次配置改动引发生成流程失败。表面症状是 401，随后又暴露出写库异常。过程涉及配置读取、请求链路、数据库 schema 不一致等多点位。

## 关键时间线
- 现象一：生成接口返回 401
- 排查一：验证 Key 在第三方接口可用
- 排查二：确认后端读取到的配置与预期一致
- 现象二：401 消失后出现 500
- 根因二：SQLite 写入失败，`flashcards.tts_script` 非空约束被触发

## 根因总结
1. 配置读取与期望不一致导致请求异常
2. 数据库 schema 与模型不一致，写库时触发 NOT NULL 约束

## 解决措施
- 配置读取链路增加可观测性，确保实际读取值可验证
- 补齐模型字段并在迁移逻辑中增补缺失列
- 清理测试插入的占位数据，保证页面展示正常

## 教训与行动项
- 一处小改动可能牵动链路多点，排查应以“端到端最小可验证闭环”为主
- 将“配置读取结果”“请求链路”“写库成功”作为三段式健康检查
- 将数据库 schema 变更固定在迁移逻辑中，避免运行时隐性失败
